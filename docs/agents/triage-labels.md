# Triage labels

Triage labels for this repo.

<!-- Default mapping: each role's label equals its canonical name. Edit the Label column if your tracker uses different strings. -->

The five canonical triage roles map to these labels in this repo:

| Role | Label | Meaning |
| --- | --- | --- |
| Needs evaluation | `needs-triage` | A maintainer needs to evaluate this issue |
| Waiting on reporter | `needs-info` | Blocked waiting for more information from the reporter |
| AFK-ready | `ready-for-agent` | Fully specified; an agent can pick it up with no human context |
| Needs human | `ready-for-human` | Needs human implementation |
| Won't fix | `wontfix` | Will not be actioned |

## Usage

When the `triage` skill processes an issue, it applies the label in the **Label** column above for the corresponding role. If you change a label string here, update it everywhere the issue tracker uses it too.
